# 11.1 Relay client implementation

Leaf node (max_children=0) — solo execution, no spawning.

## Scope

Build `src/lindenmayer/core/relay.py` (asyncio client on `websockets`,
NIP-01 + NIP-29 + NIP-42 minimum contract) and `tests/test_relay.py` plus a
mock-relay test helper. Nothing else under `src/` or `tests/` is touched.

## Design

**`RelayClient`** (asyncio, one connection per instance):

- `__init__(relay_url, keypair, config)` — stores `CoreConfig` for capability
  checks; no connection yet.
- `connect()` / `close()` / `async with` context manager — opens the
  `websockets` connection, starts a background reader task that dispatches
  incoming frames by type (`EVENT`, `EOSE`, `OK`, `NOTICE`, `CLOSED`, `AUTH`)
  to per-subscription queues / OK-waiters / an auth-challenge future.
- `publish(event, group=None)` — adds `h` tag if `group` given (raise if the
  event already carries an `h` tag conflicting), sends `["EVENT", event_dict]`,
  awaits the matching `OK` via a future keyed by event id, raises
  `RelayRejection` on `ok=False` (fail loud — never swallow). On an `auth-required:`
  NOTICE/OK prefix (NIP-42 convention) with no prior successful auth, wait for
  the `AUTH` challenge, respond, then retry the publish once.
- `query(filters, group=None)` — adds `h` tag to each filter dict if `group`,
  sends `["REQ", sub_id, *filters]`, collects verified `EVENT`s into a list
  until `EOSE`, sends `["CLOSE", sub_id]`, returns the list. Invalid-signature
  events are dropped and counted (`self.dropped_invalid`), never yielded.
- `subscribe(filters, group=None)` — same REQ setup but returns an async
  generator that yields verified events indefinitely (post-EOSE too) until
  `unsubscribe`/generator close, which sends CLOSE.
- Auth: `_handle_auth_challenge(challenge)` builds kind-22242 event
  (`relay`, `challenge` tags), signs via keypair, publishes it (bare EVENT +
  OK wait, no group), resolves the pending auth future/state.
- **Private gating**: `require_private(group)` (or a `private=True` kwarg on
  publish/subscribe/query) raises `PrivateCapabilityError` unless
  `config.has_capability(CAP_PRIVATE_READ_GATING)`. Non-private calls
  (the default) are unaffected — this is an explicit opt-in surface, not a
  wrapper around every call.
- Reconnect: bounded retry loop with exponential backoff on connection drop;
  on reconnect, re-issue `REQ` for all live subscriptions (no replay of
  missed events — no persistent queue, documented in the module docstring
  and `README.md`... actually README.md is core's, out of my ownership, so
  document only in relay.py's own docstring).

Exceptions: `RelayError` (base), `RelayRejection` (OK false), `PrivateCapabilityError`.

## Mock relay (tests/relay_mock.py, or similar — my own file, no naming clash)

An in-process `websockets.serve` server:
- EVENT -> validates minimally, stores in a dict, replies OK true (config:
  optionally OK false to test rejection).
- REQ -> filters stored events (kind/authors/kinds/#h tag match), sends
  EVENT... then EOSE; keeps subscription alive for live fan-out on later
  EVENT publishes matching the filter.
- CLOSE -> drops subscription.
- Configurable auth-required mode: rejects ops with `["OK", id, false,
  "auth-required: ..."]` / `["CLOSED", sub_id, "auth-required: ..."]` until
  the connection sends a valid signed kind-22242 AUTH event matching a
  challenge the mock sent via `["AUTH", challenge]` on connect.

## Test coverage (tests/test_relay.py)

1. publish -> OK round-trip (success and explicit rejection raises).
2. query collects stored events until EOSE.
3. subscribe yields live-published events.
4. NIP-42 auth flow: mock demands auth, client authenticates, retried op
   succeeds.
5. Invalid-signature inbound event is dropped, not delivered (assert dropped
   counter increments / event never surfaces in query or subscribe results).
6. `h`-tag scoping: group param adds `h` tag on publish and on filters; mock
   asserts/filters on it.
7. Private-gating: `require_private`/`private=True` raises
   `PrivateCapabilityError` without attestation; succeeds (or at least
   doesn't raise) with a `CAP_PRIVATE_READ_GATING` attestation in `CoreConfig`.

## Post-Mortem

Delivered as planned: `RelayClient` (connect/publish/query/subscribe, NIP-42
auth-challenge with retry-after-auth, `h`-tag group scoping, fail-loud
`PrivateCapabilityError` gating, bounded reconnect+resubscribe) plus
`tests/relay_mock.py` and 12 tests in `tests/test_relay.py`. Full suite (64
tests) green on first implementation pass. One design refinement from the
plan: `publish(event, group=...)` validates the event already carries a
matching `h` tag rather than mutating it — Event is a frozen, already-signed
dataclass, so rewriting tags post-signature would invalidate the id/sig;
`query`/`subscribe` filters (plain dicts) are safely mutated to add `#h`.
No open items; ready to commit and finish.
